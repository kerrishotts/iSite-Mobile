Project News

Don't forget to check out the project on GitHub to download the framework and see the most recent, nail-bitingly gritty happenings.

January 12, 2012

  • Big changes are a-foot...
  • Changed all .php extensions to .html. Removed all .php files, as they were unnecessary. Entire demo now uses AJAX.
  • Tweaked CSS and added support for INPUTs, SELECTs, and TEXTAREAs.
  • Added a Text Entry example.
  • Added caching via cache.manifest. Current behavior is to load all content from the server. Will change later.
  • Fixed iScroll 4 behavior regarding SELECTs, INPUTs, and TEXTAREAs.
  • Added a "Content" example, with the Bible as the source content. This is a black-on-white content area with a black bar anchored to the bottom containing Bible, Book, and Chapter SELECTs. Uses the Biblia APIs to provide a very simple Bible application proof-of-concept.
  • Added a .htaccess file to specify the type of the manifest file.

January 4, 2012

  • Significant improvements.
  • Fixed bad scrolling behavior. Some pages scrolled incorrectly, or not at all.
  • Added Very simple Twitter, Google+, and Podcast libraries to replace the old PHP(server)-based ones.
  • Menu is now loaded dynamically at startup. Could be used to change the menu on-the-fly.
  • Tweaked styling.
